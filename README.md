# PyLot

Python 脚本调度桌面管理系统，适合需要定期运行多个 Python 脚本的场景。

## 功能

- 按分类管理 Python 脚本，支持 script 和 module 两种启动模式，自动识别虚拟环境解释器
- 实时日志输出（UTF-8，支持中文）
- cron 定时调度，脚本和工作流均可设置；支持在定时任务总览直接新增/编辑、按最近运行时间筛选排序、查看最近运行情况和脚本历史日志
- 全局 .env + 脚本私有环境变量双层合并
- 卡死超时检测与强杀
- **Workflow**：拖拽编排多脚本依赖关系，支持并发执行和错误终止
- **服务管理**：管理长期运行的本地命令/服务，支持启动、停止、重启、编辑、实时日志、状态追踪
- **服务跟随 PyLot 启动**：服务可单独开启“跟随 PyLot 启动”，软件启动后自动拉起对应服务
- **飞书通知**：脚本或工作流执行结束后通过 lark-cli 发送飞书消息
- 关闭窗口最小化到系统托盘（右键菜单：定时任务 / 退出），有脚本运行时图标变色
- 深色/浅色主题切换，字体可选，窗口大小记忆
- 单实例运行（命名 Mutex 防多开）

## 技术栈

Go + [Wails v2](https://wails.io) + Vue3 + SQLite（纯 Go 驱动，无需 GCC）

## 开发

```bash
# 安装依赖
go mod download
cd frontend && npm install && cd ..

# 开发模式（热重载）
wails dev

# 打包
wails build -platform windows/amd64 -ldflags "-H windowsgui"
```

详细环境搭建见 [CLAUDE.md](./CLAUDE.md)。

## 要求

- Windows 10/11
- WebView2（系统自带）
- Python 解释器路径在 UI 中配置
- 飞书通知可选：需安装 `lark-cli` 并在设置页填入路径和 Open ID

## 服务管理

服务用于托管需要长期运行的本地进程，例如本地 API、爬虫守护进程、数据同步服务或前端开发服务。服务命令支持 Windows 命令行引号解析，工作目录可作为相对路径基准目录。

在“服务”页可以：

- 新增服务：填写名称、命令、工作目录，并选择是否跟随 PyLot 启动
- 控制服务：启动、停止、重启、删除
- 查看状态：运行状态、PID、启动时间、停止时间、最近错误
- 查看日志：服务运行时 stdout/stderr 实时输出，最近日志会保留在本次 PyLot 会话内
- 切换自启：在服务详情中直接打开或关闭“跟随 PyLot 启动”

## 定时任务总览

“定时任务”页用于集中查看和维护脚本/工作流调度规则。任务列表默认显示即将运行的启用任务，并按下次运行时间从近到远排序；可通过“今日运行”“已停止”“全部”筛选快速定位不同状态的规则。

页面下半部分显示最近运行情况，脚本和工作流会合并成一条时间线；选择脚本运行记录后，可直接查看该次运行保存的日志输出。

## 飞书通知配置

1. 安装 lark-cli：`npm install -g @larksuiteoapi/lark-cli`
2. 初始化：`lark-cli config init`（填入 App ID 和 App Secret）
3. 在 PyLot 设置页填入 lark-cli 路径和你的 Open ID
