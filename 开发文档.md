## 四、数据库建表提示词

```
在 internal/db/database.go 中实现 SQLite 初始化，建以下5张表：

1. global_config（全局配置表）
   - id INTEGER PRIMARY KEY
   - env_file_path TEXT        -- 共用.env文件路径
   - updated_at DATETIME

2. scripts（脚本基础表）
   - id INTEGER PRIMARY KEY AUTOINCREMENT
   - name TEXT NOT NULL
   - category TEXT NOT NULL    -- 枚举：crawler / processor / tool
   - interpreter_path TEXT     -- Python解释器绝对路径
   - work_dir TEXT             -- 工作目录
   - script_path TEXT          -- 脚本文件路径
   - launch_mode TEXT          -- 枚举：script / module
   - fixed_args TEXT           -- 固定启动参数
   - private_env TEXT          -- 私有环境变量，JSON格式存储 {"KEY":"VALUE"}
   - timeout_seconds INTEGER   -- 卡死超时时间（秒），0表示不检测
   - created_at DATETIME
   - updated_at DATETIME

3. schedules（定时任务表）
   - id INTEGER PRIMARY KEY AUTOINCREMENT
   - script_id INTEGER         -- 关联scripts.id
   - cron_expr TEXT            -- cron表达式
   - enabled INTEGER           -- 0/1
   - created_at DATETIME

4. run_records（任务运行记录表）
   - id INTEGER PRIMARY KEY AUTOINCREMENT
   - script_id INTEGER
   - started_at DATETIME
   - ended_at DATETIME
   - status TEXT               -- running / success / error / killed / timeout
   - log_output TEXT           -- 完整日志
   - is_error INTEGER          -- 0/1
   - env_snapshot TEXT         -- 本次加载环境摘要 JSON
   - created_at DATETIME

5. running_tasks（全局任务状态表）
   - script_id INTEGER PRIMARY KEY
   - pid INTEGER
   - started_at DATETIME

数据库文件路径为 exe 同目录下的 script-manager.db
使用 database/sql + go-sqlite3 驱动
程序启动时自动建表（IF NOT EXISTS）
```

## 五、后端核心模块提示词

### 5.1 环境加载模块

```
在 internal/env/loader.go 中实现环境变量加载：

功能：
1. LoadGlobalEnv(envFilePath string) (map[string]string, error)读取指定路径的.env文件，解析所有 KEY=VALUE，返回 map文件不存在时返回 error

2. MergeEnv(globalEnv map[string]string, privateEnv map[string]string) []string
   合并两层环境变量，privateEnv 同名 key 覆盖 globalEnv
   返回 os.Environ() 格式的 []string（"KEY=VALUE"）同时继承当前进程的系统环境变量作为基础层

3. BuildEnvSnapshot(globalEnv, privateEnv map[string]string) string
   生成环境摘要 JSON 字符串，明文记录所有 key=value，用于日志溯源

使用 github.com/joho/godotenv 解析.env文件
```

### 5.2 脚本运行模块

```
在 internal/script/runner.go 中实现脚本进程管理：

数据结构：
type RunTask struct {
    ScriptID      int
    InterpreterPath string
    ScriptPath    string
    WorkDir       string
    LaunchMode    string   // "script" or "module"
    Args          string   // 最终使用的参数（临时参数或固定参数）
    Env           []string // 合并后的环境变量
    TimeoutSecs   int      // 0=不超时
}

功能：
1. StartScript(task RunTask, logCallback func(line string, isError bool)) (pid int, err error)
   - 根据 LaunchMode 构建命令：
     script模式：interpreter_path script_path args
     module模式：interpreter_path -m script_path args
   - 设置 WorkDir 和 Env
   - 用 goroutine 分别读取 stdout 和 stderr
   - 每读到一行调用 logCallback，stderr 的 isError=true
   - 记录最后一次有输出的时间，用于卡死检测
   - 将 pid 写入 running_tasks 表
   - 返回 pid

2. StopScript(pid int)
   强杀进程：windows 下用 taskkill /F /PID

3. 卡死检测：
   启动一个 goroutine，每10秒检查一次距最后输出时间
   超过 TimeoutSecs 则调用 StopScript，触发超时回调

进程退出后：
   - 从 running_tasks 表删除记录
   - 更新 run_records 状态
   - 非0退出码触发异常回调
```

### 5.3 调度模块

```
在 internal/scheduler/scheduler.go 中实现定时调度：

使用 robfig/cron/v3，WithSeconds() 选项不需要，标准5位cron即可

功能：
1. Init() 初始化 cron 实例，程序启动时调用，从数据库加载所有 enabled=1 的定时任务注册到 cron

2. AddJob(scheduleID int, cronExpr string, scriptID int, runFunc func(scriptID int))
   注册一条定时任务，触发时调用 runFunc
   保存 cron EntryID 与 scheduleID 的映射关系

3. RemoveJob(scheduleID int) 移除定时任务

4. GetNextRunTimes() []ScheduleInfo
   返回所有已注册任务的下次运行时间ScheduleInfo { ScriptID, ScriptName, CronExpr, NextRun time.Time, Enabled bool }

5. Start() / Stop() 启动和停止 cron
```

## 六、Wails 后端绑定提示词

```
在 app.go 中实现所有暴露给前端的方法，使用 Wails 的 ctx 绑定：

全局配置：
- GetGlobalConfig() GlobalConfig
- SaveGlobalConfig(config GlobalConfig) error

脚本管理：
- GetScripts() []Script
- GetScriptsByCategory(category string) []Script
- CreateScript(script Script) (int, error)
- UpdateScript(script Script) error
- DeleteScript(id int) error

运行控制：
- RunScript(scriptID int, tempArgs string) error
  tempArgs 为空字符串时使用固定参数，非空则完全替换
- StopScript(scriptID int) error
- GetRunningScripts() []int  返回正在运行的 scriptID 列表

日志：
- GetLatestLog(scriptID int) RunRecord
- GetRunHistory(scriptID int) []RunRecord  返回最近20条记录摘要（不含完整日志）
- GetRunDetail(recordID int) RunRecord     返回完整日志

定时任务：
- GetSchedules() []Schedule
- SaveSchedule(schedule Schedule) error
- DeleteSchedule(id int) error
- ToggleSchedule(id int, enabled bool) error
- GetScheduleOverview() []ScheduleInfo    返回下次运行时间列表

实时日志推送：
使用 Wails 的 runtime.EventsEmit 推送事件到前端
事件名：
- "log:line"  payload: { scriptID, line, isError bool, timestamp }
- "task:status" payload: { scriptID, status }  status: running/success/error/timeout/killed
- "task:alert"  payload: { scriptID, scriptName, reason }  触发桌面弹窗

异常弹窗：
使用 wailsruntime.MessageDialog 弹出系统对话框
```
